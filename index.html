The Universal Winning Formula

Patterns present across every accepted paper we've analyzed.

01

Quantifiable Problem Opening

Open with a real, measurable problem — a statistic, a gap, a hard constraint. Never vague motivation.

e.g. "harvest area down 2.45% year-over-year"
02

Explicit Research Gap

State the gap directly: "while X has been studied, there is a gap in Y." The Best Paper does this with one dedicated paragraph.

Best Paper strength
03

Justified Design Choices

Every component selection backed by data or literature — never "we chose X because we thought it was good."

04

Comparative Analysis

Show at least 2 approaches, materials, or methods side-by-side. Alternatives considered and rejected with reasons.

Table or side-by-side figure
05

Multi-Level Evidence

Validate through multiple channels: theory + simulation + hardware test, or design + user study + metrics.

06

Defined Success Criteria

State the target upfront (e.g., descent rate ≤ 20 m/s, success rate ≥ 75%) and explicitly report whether it was met.

07

Honest Limitations + Root Cause

Name failures and explain why they're acceptable. "Parachute late inflation caused by heatshield blocking upward airflow."

08

Result Visualization — Not Just Tables

Every accepted paper has charts/graphs alongside tables. Raw data visualized, not just numbers in text.

Most common gap

Figure Strategy

The combination of figure types present in every accepted IES paper.

Figure Type Purpose Heatshield ✦ Thermal ✦ AR Batik AWS IoT
System Architecture Shows all components and flow ✓✓✓✓
Technical Drawing / Diagram Dimensioned, annotated design detail ✓✓✓✓
Comparative Visualization Same metric across multiple options ✓✓✓✓
Results Chart / Graph Raw data plotted, not just in table ✓✓✓✓
Hardware / Implementation Photo Proof the system was actually built ✓—✓✓
✦ Best Paper  |  Note: Thermal (Best Paper) has no hardware photo — FEM simulations serve as the visual proof instead.
💡
Key insight: Figures are never decorative. Each one has a specific analytical job — validate design, compare alternatives, show results, or prove implementation. Papers with only tables and no charts are consistently weaker.

About WinningWriter

WinningWriter is a pattern-extraction tool built from real accepted and best-paper submissions at academic conferences. Rather than generic writing advice, every pattern here is sourced directly from papers that reviewers actually accepted.
